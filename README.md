#### 1 시스템 보안의 기본 원리는 무엇인가요? : `기무가`

- 기밀성 : 허가받은 사람만 정보를 이용할 수 있으며, 그 외 사람은 접근하지 못하게 하는 성격

- 무결성 : 정보가 변환/전송/저장 중, 그 이후에도 동일함을 유지

- 가용성 : 정보가 필요한 시점에 적절히 사용할 수 있는 상태

#### 3 리눅스 시스템에서 덤프 파일을 생성하는 방법에는 무엇이 있는지 설명하세요.

- gcore 명령어 사용 : `gcore -o core.<pid> <pid>`
- gdb 사용
  ```shell
  gdb -p <pid>
  (gdb) generate-core-file core.<pid>
  (gdb) quit
  ```

- /proc/sys/kernel/core_pattern 설정 변경 : `echo "/var/coredumps/core.%e.%p.%h.%t" | sudo tee /proc/sys/kernel/core_pattern`

- unlimit 설정 : `ulimit -c unlimited`

- systemd-coredump 사용
  ```shell
  coredumpctl list
  coredumpctl dump <pid>
  ```

- 프로그램 내부에서 직접 생성
  ```shell
  #include <stdlib.h>

  int main() {
      abort();  // This will generate a core dump if allowed by ulimit
      return 0;
  }
  ```

#### 5 DB 트랜잭션이 무엇인지 설명하세요. : `ACID`

- DBMS에서 실행되는 하나의 작업 단위를 가리킨다. 

- Database의 일관성과 무결성을 유지하는 데 중요한 역할을 한다.

- ACID를 만족해야 한다

  - 원자성 : 트랜잭션의 모든 연산이 완전히 완료되거나 전혀 수행되지 않아야 함
  - 일관성 : 트랜잭션 실행 전/후의 데이터베이스 상태가 일관성을 유지해야 함
  - 격리성 : 동시에 실행되는 트랜잭션이 서로의 연산에 영향을 미치지 않아야 한다.
  - 지속성 : 트랜잭션이 완료되면 영구적으로 DB에 저장되어야 한다.

#### 6 랜섬웨어에 감염됐을 때 대처법은 무엇인가요?

1. 네트워크 연결을 차단하고, 시스템을 격리한다. 

2. 시스템을 종료하고, 랜섬 노트(해커가 남긴 요구사항과 연락 방법)을 파악한다.

3. 백업 파일이 안전한지 확인하고, 백업을 통해 데이터를 복구한다.

4. 신뢰할 수 있는 안티바이러스 소프트웨어를 사용해 감염된 파일을 스캔하고 제거한다. 감염이 심각한 경우 운영체제를 다시 설치하는 것이 최선일 수도 있다.

5. 사이버 보안 관련 기관에 침해 사고를 보고한다.

6. 정기적으로 중요한 데이터를 백업하고, 안전한 곳에 보관한다. 

#### 7 ARP 스푸핑에 대해 설명해주세요.

- 네트워크 상에서 ARP(Address Resolution Protocol, 주소 결정 프로토콜)을 악용해 자신의 MAC 주소를 다른 컴퓨터의 IP 주소에 대응시키는 공격 기법
  
- ARP 스푸핑은 `트래픽 가로채기`, `트래픽 변조`, `서비스 거부 공격(DoS)`를 일으킬 수 있다.

#### 11 MySQL에서 max_connections 설정 방법을 설명하세요.

1. MySQL 설정파일 (my.cnf or my.ini) 수정

    - Linux : `/etc/my.cnf` or `/etc/mysql/my.cnf`
    
    - Window : `C:\ProgramData\MySQL\MySQL Server X.Y/my.ini`

    설정 파일을 열어 max_connections 옵션을 추가/수정, 이후 MySQL 서버 재시작

2. MySQL 명령어를 사용한 동적 변경

    ```MySQL
    SET GLOBAL max_connections = 200;
    SHOW VARIABLES LIKE 'max_connections';
    ```

#### 13 대칭키, 비대칭키 암호화 방식에 대해 설명해주세요.

- 대칭키 암호화 
  
  - 암호화와 복호화에 동일한 키를 사용, 키가 노출되면 쉽게 해독될 수 있다
  - 비대칭키 암호화보다 속도가 빠르고 효율적
  - 데이터의 대량 전송에 사용

    ex) AES, DES

- 비대칭키 암호화
    
  - 공개키와 비공개키 쌍을 사용 
  - 대칭키 암호화보다 속도가 느리고 연산 비용이 높다
  - 키교환, 인증, 디지털 서명에서 사용
  
    ex) RSA, DSA

7. 평가 및 보고 : 대응 과정 평가 및 개선, 보고서를 작성한다.
